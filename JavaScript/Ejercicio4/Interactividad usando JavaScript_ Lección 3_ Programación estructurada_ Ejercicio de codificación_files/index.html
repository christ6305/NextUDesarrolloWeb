Las operaciones matemáticas básicas o aritméticas son muy importantes en la programación, ya que la mayoría de los sistemas de información requieren hacer cálculos basados en números para entregar un resultado al usuario.
Por otro lado, las estructuras condicionales nos sirven para guiar los procesos de nuestro sistema, basado en las acciones o datos que son ingresados por el usuario. Éstas son fundamentales para el desarrollo de una aplicación, independiente del lenguaje de programación que se utilice, pues siempre habrá situaciones en las que tendrás que evaluar el valor de una variable o dato, para tomar una decisión acerca del siguiente paso a realizar en el programa.

Por último, tenemos los ciclos. Estos son otras estructuras que nos permiten repetir un conjunto de acciones, un número de veces definido o indefinido. Generalmente son usadas dentro de los lenguajes de programación para recorrer estructuras de datos de tipo Arreglo, como los vectores y matrices.

Para realizar este ejercicio, supongamos que: estás trabajando para un supermercado, en el área de sistemas, y te piden desarrollar una aplicación para calcular el precio de un pedido, con base a los precios de todos los productos, las cantidades y el IVA.
Los otros desarrolladores te han ayudado con el HTML y te han dejado listo, en un archivo JavaScript, un objeto de datos con la información de un pedido para simular un pedido real.
Además de esto, la aplicación debe mostrar una alerta cuando el valor total del pedido sobrepase los 100.000, ya que el supermercado entrega un premio a los clientes que realizan compras mayores a este valor.

Instrucciones
  1. Descarga el codBase.zip que contiene la página web.
  2. En el archivo index.js está definido un objeto llamado “Pedido”, una variable llamada “IVA” y una variable “Resultado” inicializada en 0, que funciona como acumulador del resultado. Debes usar estas variables para realizar las operaciones.
  3. El objeto “Pedido” contiene 3 arreglos: los productos, los precios y las cantidades. Cada posición tiene relación con cada arreglo, es decir, si la posición cero de los productos corresponde a una manzana, la posición cero de los precios equivale al precio de la manzana y la posición cero del arreglo de cantidades corresponde a la cantidad de manzanas.
  4. Utilizando sólo un ciclo de tipo “for in”, almacena en la variable el resultado de sumar y multiplicar correctamente los valores por las cantidades.
  5. En otra variable llamada “valor total”, almacena el resultado de aplicar el IVA a la suma que se obtuvo de todo el pedido.
  6. Crear una variable llamada “Premio” y utilizar una condición para evaluar si el valor total sobrepasa los 100.000. De ser así, dale como valor a la variable “tiene derecho a un premio” de lo contrario asignarle “no tiene derecho a un premio”.
  7. En el archivo base hay un evento click para el botón de procesar compra, dentro de este evento debes ejecutar una alerta mostrando el siguiente mensaje: “Señor(a) { nombreUsuario }, el valor total de su pedido es { valor sin iva } y aplicando el iva el total es: { valor Total } y debido al valor de tu compra: { premio }” reemplazando las llaves por las variables correspondientes.
Buenas Prácticas
  • Utilizar un sólo ciclo para recorrer múltiples arreglos simultáneamente puede optimizar mucho tu código, reduciendo el número de procesos que debes realizar para tratar los datos. Recuerda que esto solo aplica cuando recorres múltiples arreglos del mismo tamaño.
